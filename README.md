# Gamelingo

게임을 하면서 영어 문장을 모으고, 내 문장으로 바꿔 말하는 Vite + React 앱입니다.

기본 게임인 `What Remains of Edith Finch`와 사용자가 직접 추가한 게임별 플레이 노트를 지원합니다.

## 문서

- `docs/spec.md`: Codex가 먼저 읽는 제품 범위와 데이터 계약 진입점
- `README.md`: 실행 방법, 사용 흐름, ohmesh 저장 형식, 게임 추가 위치
- `AGENTS.md`: 에이전트 검증과 커밋 규칙

## 코드 구조

- `src/App.jsx`: 라우팅, 인증 상태, 게스트/ohmesh 저장 흐름을 조율하는 앱 컨테이너
- `src/constants.js`: 저장 키, record type, 기본 게임, 기본 샘플 데이터
- `src/gameState.js`: 앱 데이터 정규화, v3 압축 저장 포맷 직렬화, 게임/노트 헬퍼
- `src/ohmeshClient.js`: ohmesh 요청, redirect, localStorage, route path 헬퍼
- `src/components/HomePage.jsx`: 게임 선택, 게임 추가/수정/삭제 UI
- `src/components/StudyBoardPage.jsx`: 게임 선택 후 보이는 공부 보드와 노트/단어/인물 모달
- `src/components/Shared.jsx`: 공통 배지와 게임 커버

## 기능 고정

### 게임 라이브러리

- 홈 화면은 게임 선택 화면입니다.
- 기본 게임은 `What Remains of Edith Finch`이고 경로는 `/games/edith-finch`입니다.
- 사용자는 `게임 추가`로 개인 게임을 만들 수 있습니다.
- 개인 게임 입력값은 `제목` 필수, `설명` 선택입니다.
- 개인 게임은 이미지 URL이나 파일 업로드를 받지 않고, 제목 첫 글자 기반 기본 커버를 표시합니다.
- 개인 게임의 경로는 `/games/{id}`입니다. 제목이나 설명을 수정해도 `id`와 경로는 유지됩니다.
- 개인 게임만 `수정`과 `삭제`가 가능합니다. 삭제 전 확인창을 띄우고, 삭제하면 해당 게임의 노트 데이터도 함께 삭제합니다.
- 기본 게임은 삭제하거나 홈 화면에서 수정할 수 없습니다.

### 공부 보드

- 게임을 선택하면 해당 게임의 공부 보드가 열립니다.
- 모든 게임은 같은 공부 보드 UI를 사용합니다.
- Edith Finch는 기본 커버, 기본 단어장, 샘플 문장 3개로 시작합니다.
- 개인 게임은 빈 문장 노트, 빈 단어장, 빈 등장인물 목록으로 시작합니다.
- 상단에는 게임 선택으로 돌아가기, 검색 표시, 저장 상태, `노트 추가`, `로그인` 또는 `로그아웃` 버튼이 있습니다.
- 보드 위 문장 노트는 모두 같은 크기의 촘촘한 카드입니다.
- 문장 노트는 자유 좌표로 움직이지 않습니다.
- 문장 노트는 왼쪽에서 오른쪽으로 한 행을 채우고, 행이 꽉 차면 다음 행의 왼쪽부터 이어지는 그리드 순서로 배치됩니다.
- 문장 노트의 오른쪽 위 핸들을 드래그해 그리드 순서를 바꿀 수 있습니다.
- 문장 노트 순서는 `sentences` 배열 순서로 저장됩니다. 자유 배치용 `x`, `y` 좌표는 새 저장 데이터에 쓰지 않습니다.
- 문장 노트 본문을 누르면 노트 모달을 엽니다.
- 노트 모달은 영어 원문을 필수로 저장하고, 한글/해석은 선택으로 저장합니다.
- 노트 모달의 `한글 숨기기` 토글은 해석 입력 영역 표시만 바꿉니다.
- 기존 문장 노트는 모달에서 삭제할 수 있습니다.
- 오른쪽 `영단어` 패널은 연필 아이콘 버튼으로 전체 단어장을 편집합니다.
- 영단어 수정 모달은 여러 행 텍스트 입력을 사용합니다.
- 영단어 수정 모달은 각 줄을 `{영어}: {번역}` 형식으로 저장합니다.
- 저장 시 빈 줄은 무시하고, 줄 순서대로 단어장을 배치합니다.
- 기존 단어와 같은 영어 항목은 기존 id를 보존하고 뜻과 위치만 갱신합니다.
- 단어를 삭제하려면 영단어 수정 모달에서 해당 줄을 지우고 저장합니다.
- 보드 위 영단어 패널은 좁은 1단 목록으로 표시하고, 여백을 줄여 많은 단어를 볼 수 있게 합니다.
- 보드 위 문장 노트와 노트 모달에는 영단어 태그를 표시하지 않습니다.
- 하단 등장인물 레일은 인물 카드를 표시하고, `등장인물 추가`로 이름과 인물 메모를 저장합니다.
- 등장인물 카드를 열면 이름과 메모를 수정하거나 삭제할 수 있습니다.

### 인증과 저장

- 로그인 전에는 게스트 모드입니다.
- 게스트 모드 데이터는 현재 브라우저의 `localStorage`에 저장됩니다.
- 게스트 모드 버튼 텍스트는 `로그인`입니다.
- `로그인`을 누르면 현재 브라우저 데이터를 pending guest save로 표시한 뒤 ohmesh 로그인으로 이동합니다.
- ohmesh 인증은 `ohmesh_session` HttpOnly 쿠키를 사용합니다. 앱 코드가 액세스 토큰을 직접 저장하거나 읽지 않습니다.
- 로그인 상태에서는 ohmesh record에 자동 저장합니다.
- 로그인 후 새 record가 없으면 `gamelingo-study-state` record를 만들고, 기존 record가 있으면 최신 record를 읽어 사용합니다.
- 로그인 직후 pending guest data가 있으면 브라우저 데이터를 ohmesh에 업로드합니다.
- 저장은 변경 후 짧게 지연된 뒤 자동으로 PATCH 됩니다.
- 저장 상태는 `불러오는 중`, `저장 중`, `저장됨`, `저장 오류`로 표시됩니다.
- 현재 ohmesh 세션이 gamelingo 앱용이 아니면 `앱 세션 확인 필요` 화면을 보여주고 로그아웃 또는 재확인을 제공합니다.
- ohmesh 확인이나 저장에 실패해도 게스트 저장은 계속 사용할 수 있습니다.

## 사용 방법

1. 홈 화면에서 기본 게임을 고르거나 `게임 추가`로 개인 게임을 만듭니다.
2. 로그인 전에는 게스트 모드로 이 브라우저에만 저장됩니다.
3. 상단의 `로그인`을 누르면 ohmesh로 로그인하고 현재 브라우저 노트를 계정에 저장합니다.
4. 게임별 공부 페이지는 `/games/edith-finch`나 `/games/custom-...`처럼 고유 URL을 가집니다.
5. 게임 중 짧은 영어 문장을 발견하면 `노트 추가`에 원문과 필요한 해석을 입력합니다.
6. 보드의 문장 카드는 왼쪽에서 오른쪽으로 정렬되고, 핸들을 드래그해 순서를 바꿉니다.
7. 오른쪽 `영단어` 패널의 연필 아이콘을 눌러 `{영어}: {번역}` 줄 목록으로 단어장을 저장합니다.
8. 하단 인물 레일에는 선택한 게임의 등장인물 이름과 짧은 메모를 추가합니다.
9. 개인 게임은 홈 화면에서 제목과 설명을 수정하거나 확인 후 삭제할 수 있습니다.

모든 문장을 저장하려고 하지 않아도 됩니다. 한 번 플레이할 때 문장 3개면 충분하고, 긴 문장보다 내가 실제로 쓸 수 있는 짧은 문장이 좋습니다.

## ohmesh 데이터

로그인 전 노트는 브라우저 `localStorage`에 압축 저장됩니다. `로그인`을 누르면 ohmesh 로그인을 거쳐 현재 브라우저 노트를 계정에 저장합니다. ohmesh 인증은 `ohmesh_session` HttpOnly 쿠키만 사용하며, 앱 코드에서 토큰을 직접 다루지 않습니다.

- 앱 슬러그: `gamelingo`
- 저장 record type: `gamelingo-study-state`
- record data: 개인 게임 목록과 게임별 미션 체크, 단어장, 단어 보드 메모, 스토리 메모, 문장 노트, 등장인물
- 게스트 저장 key: `gamelingo:v3:app:guest`
- pending guest save key: `gamelingo:v3:pendingGuestSave`
- 레거시 record type fallback: `edith-finch-study-state`
- 레거시 guest key fallback: `gamelingo:v2:edithFinch:guest`

record data는 저장 크기를 줄이기 위해 압축 포맷으로 저장합니다. 기본 단어장, 기본 샘플 문장, 체크되지 않은 미션, 빈 개인 게임 노트는 저장하지 않습니다.

```js
{
  v: 3,
  g: [
    {
      id: "custom-game-lx9p2",
      title: "Portal 2",
      description: "퍼즐 대사를 내 문장으로 바꾸기",
      createdAt: "2026-05-23T00:00:00.000Z",
      updatedAt: "2026-05-23T00:00:00.000Z"
    }
  ],
  n: {
    "edith-finch": {
      x: "오늘 자주 들은 단어를 따로 적어둔다.",
      w: "alive: 살아 있는\nattic: 다락방",
      m: "오늘은 Finch 가족과 집에 대한 기억을 따라갔다.",
      c: 39,
      s: [
        {
          i: "sentence-1",
          t: "I remember",
          h: "Opening",
          o: "I remember.",
          k: "나는 기억한다.",
          r: "I remember my first day at work.",
          p: 1
        }
      ],
      p: [{ i: "character-1", n: "Edith Finch", d: "Finch 가족의 이야기를 따라가는 화자" }]
    }
  }
}
```

상위 압축 키는 `v` 버전, `g` 개인 게임 목록, `n` 게임별 노트입니다. 노트 안의 압축 키는 `x` 단어 보드 메모, `w` 단어 key:value 리스트, `m` 스토리 메모, `c` 미션 체크 bitmask, `s` 보드 문장 노트, `p` 등장인물입니다.

문장 노트 압축 키는 `i` id, `t` 제목, `h` 챕터, `o` 원문, `k` 해석, `r` 내 문장, `p` 연습 여부입니다. 현재 화면의 보드 배치는 `s` 배열 순서만 사용하며, 레거시 데이터에 남아 있는 `x`/`y` 좌표는 읽더라도 다시 저장하지 않습니다.

기존 `edith-finch-study-state` record와 `gamelingo:v2:edithFinch:guest` localStorage 데이터는 읽기 전용 fallback으로 불러와 새 v3 구조의 `edith-finch` 노트로 마이그레이션합니다. 데이터 구조가 깨진 경우에는 앱이 멈추지 않도록 기본 데이터로 되돌리는 처리를 넣었습니다.

## 게임 추가 방식

사용자는 홈 화면의 `게임 추가`에서 개인 게임을 추가합니다. 제목은 필수이고 설명은 선택입니다. 이미지는 입력받지 않으며 앱이 제목 기반 기본 커버를 표시합니다.

코드에 기본 게임을 추가할 때는 [src/constants.js](src/constants.js)의 `defaultGames`, [src/gameState.js](src/gameState.js)의 기본 노트 데이터, 필요하면 게임 전용 기본값을 함께 수정합니다. 사용자 추가 게임은 공통 공부 화면과 빈 노트로 시작합니다.

## 문서 업데이트 규칙

- 사용자에게 보이는 기능, 버튼 이름, 저장 키, record type, 데이터 압축 키가 바뀌면 이 README와 [docs/spec.md](docs/spec.md)를 함께 갱신합니다.
- 보드 정렬 규칙이 바뀌면 `공부 보드` 섹션과 `ohmesh 데이터`의 문장 순서 설명을 같이 갱신합니다.
- 인증 또는 저장 흐름이 바뀌면 `인증과 저장` 섹션과 `ohmesh 데이터` 섹션을 같이 갱신합니다.
